개요

daero 는 GTFS 기반 국내 대중교통 door-to-door 경로탐색 API 입니다. 인증(API 키) 불필요. 모든 응답은 application/json; charset=UTF-8. CORS 전체 허용(*).

Base URLhttp://localhost:8090
인증없음 (공개)
메서드GET
시각 형식HH:MM 또는 HH:MM:SS (자정 기준, 24시 초과 = 심야)

직접 호출해보기

아래에 경로를 넣고 보내기를 누르면 실제 응답이 표시됩니다. 프리셋으로 채워볼 수 있어요.

응답이 여기에 표시됩니다.

GET /api/plan/coords

출발·도착 좌표 간 door-to-door 경로. 근처 정류장까지 도보 접근 → 대중교통 → 하차 도보를 포함하며, 추천 경로와 대안 경로들을 함께 반환합니다.

파라미터필수타입기본설명
fromLat필수double—출발 위도
fromLon필수double—출발 경도
toLat필수double—도착 위도
toLon필수double—도착 경도
time선택string08:00출발 시각 HH:MM
alternatives선택int1대안 경로 수 (0~10). 응답 options = 추천 1 + 대안 N

예시 요청:

GET /api/plan/coords?fromLat=37.5665&fromLon=126.9780&toLat=37.4979&toLon=127.0276&time=09:00&alternatives=2

GET /api/plan

정류장 ID 간 경로. 좌표 대신 정류장 ID를 직접 지정할 때 사용(/api/plan/search 로 ID 조회).

파라미터필수타입기본설명
fromStop필수string—출발 정류장 ID
toStop필수string—도착 정류장 ID
time선택string08:00출발 시각 HH:MM
alternatives선택int1대안 경로 수 (0~10)
GET /api/plan?fromStop=RS_ACC1_S-1-0150&toStop=RS_ACC1_S-1-0222&time=09:00

GET /api/plan/search

이름으로 정류장 검색. 정류장 ID·좌표를 반환합니다.

파라미터필수타입기본설명
q필수string—정류장 이름(부분 일치)
limit선택int20최대 결과 수
GET /api/plan/search?q=서울역&limit=5
→ [ { "stopId": "...", "name": "서울역", "lat": 37.55, "lon": 126.97 }, ... ]

GET /api/plan/near

좌표 근처 정류장을 도보 거리와 함께 반환(모드 접두어 포함 — BS버스·RS철도/지하철 등).

파라미터필수타입기본설명
lat, lon필수double—기준 좌표
radius선택double800반경(m)
GET /api/plan/near?lat=37.5665&lon=126.978&radius=800
→ [ { "stopId": "...", "prefix": "BS", "name": "...", "walkSec": 120 }, ... ]

GET /api/gtfs/stats · /api/gtfs/timetable

적재된 GTFS 카운트(stats)와 빌드된 경로 자료구조 상태(timetable, 모드별 패턴 포함). 헬스체크·검증용.

GET /api/gtfs/stats     → { "loaded": true, "stops": 220577, "routes": 30895, ... }
GET /api/gtfs/timetable → { "built": true, "patterns": 33071, "patternsByMode": {...} }

응답 스키마 (경로 API 공통)

최상위에는 추천 경로가 펼쳐지고(하위호환), options 배열에 [추천, 대안...]이 담깁니다.

필드타입설명
foundboolean경로 발견 여부
queryMsint탐색 소요(ms)
options[]array경로 옵션 목록(추천 + 대안)
options[].tagstring추천 / 지하철·철도 우선 / 환승 최소 / 대안 …
options[].departure, arrivalstring출발·도착 시각 HH:MM
options[].durationMinint총 소요(분)
options[].transfersint환승 횟수
options[].estimatedFareKrwint예상 요금(원, 도시 통합요금 근사)
options[].modes[]string[]사용 교통수단
options[].legs[]array구간 목록
legs[].modestringBUS·SUBWAY·RAIL·AIR·FERRY·WALK
legs[].routestring노선명(도보 제외)
legs[].from, tostring승·하차 정류장명
legs[].depart, arrivestring구간 출발·도착 시각
legs[].minint구간 소요(분)
legs[].fromLat/fromLon/toLat/toLondouble구간 양끝 좌표(경로선용)
{
  "found": true, "queryMs": 140,
  "tag": "추천", "departure": "09:00", "arrival": "09:29",
  "durationMin": 29, "transfers": 1, "estimatedFareKrw": 1350, "modes": ["BUS"],
  "legs": [ { "mode": "BUS", "route": "262", "from": "서울역", "to": "종로2가",
              "depart": "09:01", "arrive": "09:07", "min": 6,
              "fromLat": 37.55, "fromLon": 126.97, "toLat": 37.57, "toLon": 126.98 } ],
  "options": [ /* {tag:"추천",...}, {tag:"지하철·철도 우선",...}, {tag:"환승 최소",...} */ ]
}

개인정보처리방침

daero(이하 "서비스")는 대중교통 경로탐색을 제공하며, 이 방침은 서비스 이용 시 개인정보 처리에 관한 사항을 안내합니다. 서비스는 이용자 계정을 두지 않으며 개인정보를 직접 저장하지 않습니다.

1. 수집·처리 항목

  • 서비스 이용 기록: 접속 로그(IP, 접속 시각, 브라우저 정보) — 서비스 운영·보안·오류 대응 목적. 경로 질의 파라미터(좌표·시각)는 처리 후 별도 저장하지 않습니다.
  • 광고 관련: 지도(웹) 화면에 카카오 애드핏 광고가 노출되며, 맞춤형 광고 제공을 위해 쿠키·광고식별자가 사용될 수 있습니다. (오픈 API 응답에는 광고·추적이 포함되지 않습니다.)

2. 쿠키 및 광고

광고 노출·측정을 위해 광고 사업자(카카오)가 쿠키를 설정할 수 있습니다. 이용자는 브라우저 설정에서 쿠키 저장을 거부·삭제할 수 있으며, 이 경우 광고 개인화가 제한될 수 있습니다. 카카오 광고 및 개인정보 처리에 대한 자세한 내용은 카카오의 정책을 참고하세요.

3. 제3자 제공·처리위탁

서비스는 광고 제공을 위해 카카오 애드핏을 이용하며, 관련 데이터는 해당 사업자의 정책에 따라 처리됩니다. 그 외 이용자 개인정보를 제3자에게 판매·제공하지 않습니다.

4. 보유·이용 기간

접속 로그는 운영·보안 목적에 필요한 기간 동안만 보관 후 파기합니다.

5. 이용자 권리

브라우저 쿠키 거부, 광고 개인화 해제 등을 통해 권리를 행사할 수 있습니다.

6. 데이터·지도 출처
